What the .... is Laravel?

MVC Web Framework For PHP

Model // View // Controller

MVC

Software Architectural Pattern

  • Originally designed for implementing user interfaces
  • Separates internal representations of information from user-presented information
    • https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93controller

    Implemented in..

    Ruby on Rails, Django, ASP.NET MVC, Express, EmberJS, AngularJS, Backbone (to some extent)

    https://en.wikipedia.org/wiki/Model%E2%80%93view%E2%80%93controller

    Why would you want to use a framework?

    Developer

  • Get shit done faster
  • Learn the best practises from a community
  • Tap into more advanced architecture easily

    • CTO / Business owner

  • Don't reinvent the wheel (wasting money)
  • Leverage the already existing community in developer training
  • Certain level of basic security, when implemented properly
  • Faster development time
  • Helps keeping the tech stack organised
  • Certain level of scalability
  • Community for debugging & scaling

    • Customer

  • Easier to find new developers / agencies to help later
  • Certain level of sanity quaranteed in the application architecture
  • Security (validation etc.)

    • Laravel Philosophy

    https://laravel.com/docs/4.2/introduction

    "Happy developers make the best code"

    - Taylor Otwell, creator of Laravel

    Goals & purpose

  • Combine the best parts of the best frameworks & communities around the planet
  • Make web development an enjoyable, creative & fulfilling experience
  • Introduce & embrace sophisticated application architecture for masses
  • Batteries included

    • Includes out of the box

  • Inversion of control container
  • Expressive database migration system
  • Tightly integrated unit testing support
  • Modular packaging system
  • Eloquent ORM (Object-relational mapping)

    • Query builder, Reverse routing, Restful controllers, Class autoloading, Blade templating engine, Database seeding, Automatic pagination....
